## Name {#name}

```
profil - execution time profile
```



### Library {#library}

```
Standard C library (libc, -lc)
```



## Synopsis {#synopsis}

```
#include <unistd.h>

int profil(unsigned short *buf, size_t bufsiz,
           size_t offset, unsigned int scale);
```


   Feature Test Macro Requirements for glibc (see [feature_test_macros(7)](feature_test_macros.html)):

```
profil():
    Since glibc 2.21:
        _DEFAULT_SOURCE
    In glibc 2.19 and 2.20:
        _DEFAULT_SOURCE || (_XOPEN_SOURCE && _XOPEN_SOURCE < 500)
    Up to and including glibc 2.19:
        _BSD_SOURCE || (_XOPEN_SOURCE && _XOPEN_SOURCE < 500)
```



## Description {#description}

```
This  routine  provides a means to find out in what areas your program spends most of its time.  The argument buf points to bufsiz bytes of core.  Every virtual 10 milliseconds, the user's program
counter (PC) is examined: offset is subtracted and the result is multiplied by scale and divided by 65536.  If the resulting value is less than bufsiz, then the corresponding entry in buf  is  inâ€
cremented.  If buf is NULL, profiling is disabled.
```



### Return Value {#return-value}

```
Zero is always returned.
```
